Dưới đây là review ngắn gọn theo góc nhìn reviewer của journal Expert Systems with Applications (ESWA) — tập trung vào novelty, methodology, evaluation, và khả năng accept.

⸻

Reviewer Comments — ESWA

1. Overall Assessment

Recommendation (preliminary): Major Revision

Proposal presents an interesting agentic RAG architecture for adaptive technical interview assessment, combining multi-agent orchestration, knowledge-gap reasoning, and differentiated retrieval strategies. The topic fits the scope of expert systems and intelligent decision-support systems.

However, in its current form the proposal still suffers from insufficient methodological rigor and evaluation realism, which are critical for a Q1 journal such as ESWA.

⸻

2. Strengths

2.1 Clear System Architecture

The ATIA architecture is well-structured:
• Multi-agent decomposition
• Response Analyst
• Knowledge Gap Analyzer
• Question Strategy Agent
• Grounded Question Generator
• Shared Interview Context Object (ICO) as coordination state
• Differentiated retrieval pipelines

This provides good system transparency and traceability, which aligns well with expert systems research.

⸻

2.2 Clear Research Questions

The proposal defines 5 concrete research questions:
• multi-agent vs monolithic
• hybrid symbolic + neural reasoning
• interview policy effectiveness
• differentiated retrieval
• robustness to candidate behaviors

This is well aligned with ablation-driven evaluation, which reviewers generally appreciate.

⸻

2.3 Ablation-Oriented Framing

The proposal implicitly allows component-level ablation:

Full system vs
• − multi-agent
• − graph reasoning
• − policy
• − differentiated retrieval

This experimental design is appropriate for architecture contribution papers.

⸻

3. Major Weaknesses

3.1 Weak Evaluation Dataset

The proposal uses:

150 simulated interviews

This is a major weakness.

ESWA reviewers typically expect:
• real human data
• or at least human evaluation

Simulated interviews generated by LLMs may introduce evaluation circularity.

Without stronger validation, results may be considered self-referential.

⸻

3.2 Metrics Are Not Fully Defined

Metrics like:
• Assessment Alignment Score
• Grounding Rate
• Probing Efficiency
• Topic Coverage

are mentioned but not formally defined.

For publication, the paper must clearly specify:
• mathematical definitions
• evaluation procedures
• annotation protocol

Otherwise reviewers will question reproducibility.

⸻

3.3 Novelty Risk

The proposal claims novelty in:
• agentic RAG
• hybrid knowledge-gap reasoning
• differentiated retrieval

However, current literature already contains many works on:
• multi-agent LLM systems
• RAG architectures
• adaptive tutoring systems

Therefore the paper must clearly formalize the novel component, for example:
• a formal knowledge-gap inference algorithm
• a decision policy model
• or a new evaluation benchmark

Otherwise reviewers may consider the work system engineering rather than research contribution.

⸻

3.4 Lack of Baseline Detail

Baselines listed:
• monolithic LLM
• vanilla RAG
• single-agent RAG
• static sequence RAG

But these are too loosely defined.

A Q1 reviewer will ask:
• Which model? (GPT-4, Claude, etc.)
• Which RAG implementation?
• What retrieval size?
• Same prompt budget?

Without strict control, comparisons may be considered unfair.

⸻

4. Minor Issues

4.1 Cost realism

Budget:

$500–650 API cost

For a reproducible study this is suspiciously small, especially if running ablations.

⸻

4.2 Benchmark scope

Only 3 technical domains:
• DSA
• System design
• Backend

This might be considered too narrow.

⸻

5. Expected Reviewer Questions

A reviewer will likely ask: 1. How is knowledge gap reasoning formally defined? 2. How are expert ground truths constructed? 3. Are simulated interviews LLM-generated? 4. How do you avoid evaluation bias? 5. Why is multi-agent needed vs single-agent with tool use?

⸻

6. Acceptance Probability (Estimated)

If submitted as currently designed:

Journal Probability
ESWA ~30–40%

Main risk:
• evaluation realism
• unclear novelty

⸻

7. Key Improvements Needed

To reach ESWA publishable level, the proposal should:

1️⃣ Add Human Evaluation

Example:
• 5–10 experienced engineers
• annotate candidate responses
• compare ATIA vs baseline

⸻

2️⃣ Formalize Knowledge Gap Reasoning

Provide:
• graph inference rule
• severity scoring function
• update algorithm

⸻

3️⃣ Define Metrics Formally

Example:

Assessment Alignment Score =
expert_label_match / total_questions

⸻

4️⃣ Clarify Baseline Implementations

Include:
• model versions
• prompt templates
• retrieval parameters

⸻

8. Final Verdict (as reviewer)

The proposal shows good system design and research direction, but to meet ESWA standards it must strengthen:
• evaluation methodology
• formal novelty
• experimental rigor

With these improvements, the work could become a solid applied AI systems paper.
